Search string

If string1 and string2 are present, the search is performed in AND mode for the entire file or DB object.
It is possible to indicate if you want to search the file system or the database (only Oracle is supported).
For the search in the database it is possible to indicate a double choice and the contents of the UT_LOV table will also be read among the scanned objects. The filters are valid only for the files searched in the file system and if more than one, they must be listed separated by a comma such as: .rdf, .fmb, .sql
Where a directory (or more) does not need to be searched, a list can be entered in the "Exclude directories" field; the program will exclude the directory wherever it is present in the tree.
By nature of filesystems, the same directory name can exist in different branches; exclusion will remove the indicated directory from all branches!
Specify output file where to write results in csv format. The results are also shown on the screen where with a double click it is possible to open the files, DB objects included.

Some notes (ICOM-SMILE):
The search in the ICOM-UNIFACE sources must be evaluated as in uniface, packages present in the SMILE DB could be used by ICOM-UNIFACE but through aliases.
In this case, no result would be obtained!

Import-Export tools

This section was created to allow you to save tables, especially for those tables that contain BLOB fields.
This section allows you to take a table from an Oracle DB and copy it into a SQLite DB residing on the PC. It can be subsequently resumed and reported on the Oracle DB (the target table must not exist). Any BLOB fields are not saved inside the SQLite DB (the limit of which would still be 1MB of space) but directly in a filesystem folder. From this same table, the import function inside Oracle will draw the attachments by reconstructing the starting structure.
